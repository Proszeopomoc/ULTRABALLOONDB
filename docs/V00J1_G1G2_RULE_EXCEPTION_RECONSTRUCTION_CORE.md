# V00J1_G1G2_RULE_EXCEPTION_RECONSTRUCTION_CORE

Scope: add a compact structural compression layer above the hot layout.

Core rule:

```text
G1 = rule/model/structure
G2 = exceptions/residual
G3 = navigation/queryable structure
G4 = recovery/patch layer
G5 = SHA256/audit validation
```

Goal:

```text
REBUILD(G1,G2,G3,G4) == ORIGINAL bytes
```

This is not a universal compressor. It must not claim wins on random,
encrypted, or already-compressed bytes. It is a structural reconstruction
system for datasets where most bytes are generated by a compact rule and only a
small residual is needed.

## Boundaries

- G1/G2 compression does not promote trust.
- G1/G2 compression does not replace the canonical archive.
- Hot layout remains for fast recall.
- Canonical archive remains the source of truth.
- Random negative controls must not be falsely reported as compression wins.

## V00J1 tests kept intentionally small

This gate avoids audit-of-audit bloat. It checks only:

1. low-exception formula matrix: strong structural compression;
2. prefix-ID family: template + exceptions;
3. deterministic random negative control: no false compression claim.

The next gates can add real datasets after the core contract is stable.
